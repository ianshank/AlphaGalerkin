## Level 2 — Containers

```mermaid
C4Container
  title Containers — claude-code-platform
  System_Boundary(ccp, "claude-code-platform repo") {
    Container(catalog, "Marketplace Catalog", "JSON", "Generated plugins array from manifests + pins; authored identity fields")
    Container(pins, "Release Pin Manifest", "JSON", "release/pins.json — tag → sha map (ADR-0003)")
    Container(p1, "eng-standards plugin", "Markdown/JSON/Python", "Skills, agents, quality hook; self-contained incl. vendored _runtime/")
    Container(harness, "Validation Harness", "Python 3.11 / pydantic (dev-side)", "Schema, parity, vendor-sync, path/import, frontmatter gates")
    Container(hooklib, "hook_runtime", "Python stdlib only", "Canonical hook I/O, JSON stderr logging, tunables, fail-safe; VENDORED per plugin")
    Container(ci, "CI Pipeline", "GitHub Actions", "ubuntu + macos matrix; pinned claude CLI; offline steps only")
  }
  Container_Ext(cache, "Plugin cache", "~/.claude/plugins/cache", "Per-machine copies of the PLUGIN DIRECTORY ONLY")
  Rel(harness, catalog, "Validates + regenerates (sync_catalog)")
  Rel(harness, pins, "Reads for catalog generation")
  Rel(harness, p1, "Validates + parity-checks vendored runtime")
  Rel(hooklib, p1, "Vendored into (build-time sync_runtime, NOT runtime import)")
  Rel(ci, harness, "Executes")
  Rel(cache, catalog, "Resolved via Claude Code install")
```

## Dynamic view — distribution & runtime

```mermaid
sequenceDiagram
  participant Dev as Consumer repo dev
  participant CC as Claude Code
  participant MP as marketplace.json (GitHub)
  participant PL as Plugin (sha-pinned source)
  Dev->>CC: open repo (trust dialog)
  CC->>MP: resolve extraKnownMarketplaces
  CC->>PL: fetch plugin dir at pinned sha, cache plugin dir only
  CC-->>Dev: prompt to enable declared plugins
  Note over CC: session runs
  CC->>PL: hook fires (PostToolUse)
  PL->>PL: stdlib-parse stdin JSON (vendored _runtime), read CCP_* env + config/defaults.json
  PL-->>CC: warn/allow (fail-safe), JSON log to stderr
```
