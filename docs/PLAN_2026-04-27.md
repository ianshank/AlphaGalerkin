## Section 5 — Engineering Discipline (cross-cutting, applies to ALL the above)

These rules apply to every PR delivered against this plan. Stated up front so reviewers can reject violations early.

| Principle | Concrete rule |
|---|---|
| **No hardcoded values** | Every constant affecting numerical behavior is a Pydantic `Field` with `description` and bounds (`gt`, `ge`, `lt`, etc.). Module-level constants only for stability floors that should never need tuning. The Noyron HX milestones already follow this — keep the bar there. |
| **Backwards compatibility** | New features are additive optional fields in existing config classes. Never break existing signatures. Use `model_validator(mode="after")` for cross-field validation. Default values must reproduce prior behavior. |
| **Reusable components** | Extend existing protocols (`DomainGeometry`, `PDEOperator`, `Evaluator`, `BaseScenario`) rather than parallel implementations. New abstractions only when the user can name three concrete reusers. |
| **TDD** | Write the test first when the contract is known. PR description must cite the test and the guarantee it enforces. Property-based tests via Hypothesis for math kernels (existing pattern). |
| **Lint/type clean before merge** | `ruff check src/ tests/`, `ruff format src/ tests/ --check`, `mypy src/ --strict` all pass on touched files. Pre-existing errors out of scope per branch. |
| **Per-module coverage gate** | 85% minimum (current default). Promoted to 90%+ on the Noyron HX surface. Enforced by `pytest --cov-fail-under` in the regression-surface CI workflow (Section 2.4). |
| **NumPy 2.x compat** | No `np.bool8`, `np.float_`, `np.int_`, `np.product`, etc. Currently clean — keep it that way. |
| **No `# type: ignore` without comment** | Every ignore must say *why* (e.g., `# type: ignore[import-not-found]  # picogk gated behind [picogk] extra`). |
